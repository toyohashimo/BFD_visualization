# 過去比較モード Mode 6 要件定義書

## 📋 概要

**グローバルモード**: 過去比較モード  
**モード番号**: 6  
**モードID**: `historical_archetype_segment_brand`  
**モード名**: アーキタイプ分析（セグメント、ブランド: X=アーキタイプ×過去比較）  
**実装日**: 2025-11-30  
**ステータス**: 📝 要件定義完了

---

## 🎯 目的

単一セグメント・単一ブランドにおけるアーキタイプ指標（12の原型）の時系列変化を可視化し、過去のデータと比較することでブランドパーソナリティの変化や進化を分析する。

---

## 📊 モード概要

### 分析軸の構成

| 軸 | 役割 | 値 | 選択方式 | 詳細 |
|----|------|-----|---------|------|
| **分析項目** | X_AXIS | 12のアーキタイプ | 固定 | ブランドパーソナリティの12原型 |
| **セグメント** | FILTER | セグメント名 | SA（単一選択） | 分析対象のセグメント |
| **ブランド** | FILTER | ブランド名 | SA（単一選択） | 分析対象のブランド |
| **データ系列** | SERIES | データソース（期間） | 複数（最大3） | 時系列データ |

### アーキタイプ指標（12の原型）

| 指標キー | 日本語名 | 英語名 | 説明 |
|---------|--------|--------|------|
| creator | 創造者 | Creator | 新しい価値を創造し、革新的なアイデアを提供 |
| ruler | 統治者 | Ruler | その分野をリードし、権威と信頼を持つ |
| sage | 賢者 | Sage | 知識と専門性を持ち、真実を追求 |
| explorer | 探検家 | Explorer | 新しい体験を求め、自由と冒険を重視 |
| innocent | 幼子 | Innocent | 純粋で誠実、シンプルさと信頼を大切にする |
| outlaw | 無法者 | Outlaw | 既存の枠を破り、革新的な変化をもたらす |
| magician | 魔術師 | Magician | 夢を現実にし、特別な体験を創造 |
| hero | 英雄 | Hero | 困難を乗り越え、勇気と力を与える |
| lover | 恋人 | Lover | 美と魅力を追求し、感性に訴える |
| jester | 道化師 | Jester | 楽しさと喜びをもたらし、人生を楽しむ |
| regular | 仲間 | Regular Guy/Gal | 親しみやすく、身近な存在として共感を得る |
| caregiver | 援助者 | Caregiver | 他者をケアし、安心と信頼を提供 |

**アーキタイプ指標の特徴**:
- 各アーキタイプは複数のブランドイメージ項目から計算される
- 12の原型すべてを同時に可視化することでブランドパーソナリティの全体像を把握
- 時系列比較により、ブランドパーソナリティの進化を追跡

## 🔄 詳細分析モードとの対応関係

### 対応する詳細分析モード

**詳細分析モード Mode 13**: `archetype_segment_brands`
- 名称: アーキタイプ分析（セグメント: X=アーキタイプ×ブランド）
- 比較軸: 複数ブランド間の比較
- X軸: アーキタイプ指標（12原型）
- データ系列: 複数ブランド

**過去比較モード Mode 6**: `historical_archetype_segment_brand`（本モード）
- 名称: アーキタイプ分析（セグメント、ブランド: X=アーキタイプ×過去比較）
- 比較軸: 単一ブランドの時系列比較
- X軸: アーキタイプ指標（12原型）
- データ系列: 複数時点データ

### 違いの明確化

| 項目 | 詳細分析モード (Mode 13) | 過去比較モード (Mode 6) |
|------|------------------------|------------------------|
| **比較対象** | 複数ブランド | 複数時点 |
| **セグメント選択** | SA（単一） | SA（単一） |
| **ブランド選択** | MA（複数） | SA（単一） |
| **データソース** | 1つ（現在） | 最大3つ（時系列） |
| **凡例表示** | ブランド名 | データソース名（期間） |
| **分析目的** | ブランド間のパーソナリティ比較 | 同一ブランドのパーソナリティ進化追跡 |

## 🔄 他の過去比較モードとの関係

### Mode 4（ブランドパワー分析①）、Mode 5（ブランドパワー分析②）との比較

| 項目 | Mode 4（現在パワー） | Mode 5（将来性パワー） | Mode 6（アーキタイプ） |
|------|---------------------|---------------------|---------------------|
| **指標数** | 4つ（BP1-BP4） | 6つ（FP1-FP6） | 12個（12原型） |
| **指標内容** | 認知・興味・購入・推奨 | 新しさ・注目度・成長性 | ブランドパーソナリティ |
| **分析観点** | 現在のブランドパワー | 将来性・成長性 | ブランドの個性・性格 |
| **デフォルトチャート** | レーダーチャート | レーダーチャート | レーダーチャート |
| **計算方法** | 直接集計 | 直接集計 | ブランドイメージから算出 |

### 共通点

- ✅ 過去比較モードの設計パターンを継承
- ✅ セグメント・ブランド共にSA（単一選択）
- ✅ データソースが系列となる
- ✅ 固定項目セット（12原型固定）
- ✅ `transformDataForHistoricalChart`をそのまま使用可能
- ✅ レーダーチャートがデフォルト

### Mode 6の特徴

1. **指標数が最多**: 12個の指標を持ち、最も多面的な分析が可能
2. **計算ベース**: ブランドイメージ項目から算出される（間接指標）
3. **定性的な評価**: ブランドの「性格」や「個性」を数値化

## 📈 グラフ仕様

### デフォルトチャートタイプ

**レーダーチャート（Radar Chart）**

### 理由

アーキタイプ分析は12の原型を総合的に把握する必要があり、レーダーチャートが最適:
- 12の原型のバランスを視覚的に把握
- ブランドパーソナリティの全体像を一目で理解
- 時系列でのパーソナリティ変化を面積で比較
- 強み・弱みの変化が一目瞭然
- 12角形のレーダーチャートで豊かな表現

### 切り替え可能なチャートタイプ

- ✅ レーダーチャート（デフォルト）
- ✅ 棒グラフ
- ✅ 折れ線グラフ

### グラフ出力例（レーダーチャート - 12角形）

```
           creator
             ↑
    caregiver |  ruler
         ↖   |   ↗
            \|/
  regular ← + → sage
            /|\
         ↙   |   ↘
     jester  |  explorer
             ↓
          lover

凡例:
━━━ 2025年6月
─ ─ 2024年6月
・・・ 2023年6月
```

**レーダーチャートの視覚的効果**:
- 12角形により、4角形（Mode 4）、6角形（Mode 5）よりも詳細な分析が可能
- ブランドの個性が形状として現れる
- 時系列での形状変化が直感的に理解できる

## 🔄 データ変換ロジック

### 既存ロジックの再利用

Mode 1-5で実装済みの`transformDataForHistoricalChart`関数をそのまま使用可能:

```typescript
// utils/dataTransforms.ts
export const transformDataForHistoricalChart = (
  dataSources: DataSource[],
  config: AnalysisModeConfig,
  selectedSegment: string,
  selectedBrand: string,
  selectedItem: string,
  labelGetters: Record<AxisType, (key: string) => string>
): ChartDataPoint[] | null
```

### データ変換の流れ

1. **アクティブなデータソースを抽出**
   ```typescript
   const activeSources = dataSources.filter(ds => ds.isActive);
   ```

2. **X軸の項目キー取得**
   ```typescript
   const xAxisKeys = config.axes.items.fixedItems;
   // ['creator', 'ruler', 'sage', 'explorer', 'innocent', 'outlaw',
   //  'magician', 'hero', 'lover', 'jester', 'regular', 'caregiver']
   ```

3. **各項目ごとにデータポイント生成**
   ```typescript
   {
     name: '創造者',
     '2025年6月': 72.3,
     '2024年6月': 69.7,
     '2023年6月': 67.1
   }
   ```

### データ取得パス

```typescript
const value = source.data[selectedSegment]?.[selectedBrand]?.[archetypeKey] || 0;
```

### アーキタイプデータの計算

アーキタイプスコアは、複数のブランドイメージ項目から算出されます:

```typescript
// 例: 創造者 (creator) の計算
// archetypeConfigs.ts から対応する項目を取得
const creatorItems = ARCHETYPE_DEFINITIONS.creator.items;
// ['新しい価値を提案する', '先見性のある', '独創性がある', '知的な', '信念・ポリシーのある']

// 各項目のスコアを平均化
const creatorScore = average(
  creatorItems.map(item => brandImageData[item] || 0)
);
```

**注意**: アーキタイプスコアの計算は、Excelパーサーまたはデータ変換処理で事前に行われていることを前提とします。

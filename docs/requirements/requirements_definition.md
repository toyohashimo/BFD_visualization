# 要件定義書 - BFD Analytics (Brand Funnel Dashboard)

## 1. はじめに

本ドキュメントは、Excel形式のブランドファネル調査データを可視化・分析するためのWebアプリケーション「BFD Analytics」の要件を定義するものである。

**バージョン**: 3.0.0-beta  
**最終更新**: 2025-11-30

## 2. プロジェクトの目的

- 複雑なExcelデータを直感的なグラフとして可視化し、ブランド分析業務を効率化する
- セグメント間比較やブランド間比較、時系列推移分析を容易に行えるようにし、マーケティング施策の立案を支援する
- クライアントサイドで完結するセキュアなデータ処理環境を提供する
- **20の分析モード（詳細分析14種 + 過去比較6種実装済み + 8種予定）**により、多角的なデータ分析を可能にする

## 3. ターゲットユーザー

- マーケティング担当者
- データアナリスト
- ブランドマネージャー
- 経営企画部門

## 4. 機能要件

### 4.1 データ入力・管理

#### 4.1.1 ファイルインポート
- **対応形式**: Excelファイル (`.xlsx`, `.xls`) および CSVファイル (`.csv`) の読み込みに対応する
- **アップロード方法**: 
  - ドラッグ＆ドロップによるアップロード
  - ファイル選択ダイアログによるアップロード
  - 自動読み込み（開発環境で `sample_202506.xlsx` を自動的に読み込む、本番環境では削除予定）
- **フォーマット仕様**: 3行目をヘッダー、4行目以降をデータ、D列をブランド名とする特定のフォーマットを解析する

#### 4.1.2 データ解析
- **ファネル指標**: 以下の6項目を読み込む
  - FT: 認知あり(TOP2)
  - FW: 興味あり(TOP2)
  - FZ: 好意あり(TOP2)
  - GC: 購入・利用意向あり(TOP2)
  - GJ: 購入・利用経験あり(TOP5)
  - GL: リピート意向あり(TOP2)

- **時系列指標**: 以下の5項目を読み込む
  - T1: 直近1ヶ月以内に購入・利用した
  - T2: 過去2～3ヶ月以内に購入・利用した
  - T3: 過去4ヶ月～半年未満に購入・利用した
  - T4: 半年～1年未満に購入・利用した
  - T5: 1年以上前に購入・利用した

- **ブランドパワー指標**: 以下の4項目を読み込む
  - BP1: 好きなブランド (複数回答)
  - BP2: 現在のパワー
  - BP3: 5年前のパワー
  - BP4: なんとなくのパワー

#### 4.1.3 匿名化モード（デモ機能）
- **概要**: 実データを隠すための「匿名化（Demo）」トグルスイッチを提供する
- **動作**:
  - ONの場合、ブランド名を「ブランド1, ブランド2...」のようにマスキングする
  - このマスキングは、画面上の表示だけでなく、CSVエクスポートされたデータにも適用される
- **起動方法**: サイドバーヘッダー領域（"BFD Analytics" ロゴ部分）をクリック
- **視覚的フィードバック**: 
  - ON時: 「DEMO」バッジと `EyeOff` アイコンを表示
  - OFF時: アイコンのみ（ホバー時）

#### 4.1.4 データ永続化
- ユーザーの選択状態（分析モード、対象ブランド、選択セグメント等）をブラウザの `localStorage` に保存し、次回アクセス時に復元する

### 4.2 分析モード

以下の**20の分析モード（詳細分析14種 + 過去比較6種実装済み + 8種予定）**を提供する。

#### 4.2.1 グローバルモード（NEW）

アプリケーション全体に適用される分析モードの切り替え。

**詳細分析モード**
- **目的**: 複数ブランド/セグメントの横断比較
- **データソース**: 単一Excelファイル
- **選択方式**: ブランド・セグメントを複数選択（最大10個）
- **分析モード数**: 14種類

**過去比較モード（NEW）**
- **目的**: 時系列データの推移分析
- **データソース**: 最大3つのExcelファイル
- **選択方式**: ブランド・セグメントは単一選択、データソースを複数選択
- **分析モード数**: 6種類（実装済み）+ 8種類（実装予定）

#### 4.2.2 詳細分析モード（14種類）

**モード1: ファネル分析（セグメント×複数ブランド）**
- **目的**: 特定のセグメント（Excelシート）を選択し、その中の複数のブランドを比較するモード
- **用途**: 競合他社との比較分析に使用する
- **X軸**: ファネル指標（固定6項目）
- **データ系列**: 選択した複数のブランド
- **フィルタ**: セグメント（単一選択）

**モード2: ファネル分析（ブランド×複数セグメント）**
- **目的**: 特定のブランドを選択し、複数のセグメント（Excelシート）間での数値を比較するモード
- **用途**: 属性別（性年代別など）のブランド浸透度分析に使用する
- **X軸**: ファネル指標（固定6項目）
- **データ系列**: 選択した複数のセグメント
- **フィルタ**: ブランド（単一選択）

**モード3: ファネル分析（項目×複数ブランド）**
- **目的**: 特定のファネル指標を選択し、複数のブランドを複数のセグメントで比較するモード
- **用途**: 特定指標（例: 認知度）に焦点を当てた多角的比較
- **X軸**: 選択した複数のブランド
- **データ系列**: 選択した複数のセグメント
- **フィルタ**: ファネル指標（単一選択）

#### 4.2.2 時系列分析カテゴリー（3モード）

**モード4: 時系列分析（ブランド×複数セグメント）**
- **目的**: 特定のブランドの時系列項目を複数のセグメント間で比較するモード
- **用途**: 購入時期の分布をセグメント別に分析
- **X軸**: 時系列指標（固定5項目）
- **データ系列**: 選択した複数のセグメント
- **フィルタ**: ブランド（単一選択）

**モード5: 時系列分析（セグメント×複数ブランド）**
- **目的**: 特定のセグメント内で複数のブランドの時系列項目を比較するモード
- **用途**: 直近利用者の割合をブランド間で比較
- **X軸**: 時系列指標（固定5項目）
- **データ系列**: 選択した複数のブランド
- **フィルタ**: セグメント（単一選択）

**モード6: 時系列分析（項目×複数ブランド）**
- **目的**: 特定の時系列項目を選択し、複数のブランドを複数のセグメントで比較するモード
- **用途**: 特定時期（例: 直近1ヶ月）の利用状況を多角的に比較
- **X軸**: 選択した複数のブランド
- **データ系列**: 選択した複数のセグメント
- **フィルタ**: 時系列指標（単一選択）

#### 4.2.3 ブランドイメージ分析カテゴリー（2モード）

**モード7: ブランド分析（セグメント×複数ブランド）**
- **目的**: 特定のセグメント内で、複数のブランドについて、ブランドイメージ項目（全133種）の中から上位30項目を抽出し、ブランド間でのイメージ比較を行うモード
- **用途**: ブランドイメージの強み・弱みの可視化、競合他社とのイメージポジショニング比較
- **X軸**: ブランドイメージ項目（自動選定、TOP30）※先頭ブランドの降順で自動抽出
- **データ系列**: 選択した複数のブランド
- **フィルタ**: セグメント（単一選択）
- **データソース**: Excelシート AP列～FS列（計133項目）
- **特記事項**: 
  - 「あてはまるものはない」は除外して並べ替え
  - X軸ラベルは縦方向に配置（項目数が多いため）
  - 選択ブランドの先頭（一番上）が基準ブランドとなる

**モード8: ブランド分析（ブランド×複数セグメント）**
- **目的**: 単一ブランドについて、複数のセグメント間で、ブランドイメージ項目（全133種）の中から上位30項目を抽出し、セグメント間でのイメージ比較を行うモード
- **用途**: セグメント別のブランドイメージ分析、属性別のイメージの違いの把握
- **X軸**: ブランドイメージ項目（自動選定、TOP30）※先頭セグメントでの基準ブランドの降順で自動抽出
- **データ系列**: 選択した複数のセグメント
- **フィルタ**: ブランド（単一選択）
- **データソース**: Excelシート AP列～FS列（計133項目）
- **特記事項**: 
  - 「あてはまるものはない」は除外して並べ替え
  - X軸ラベルは縦方向に配置（項目数が多いため）
  - 選択セグメントの先頭（一番上）が基準セグメントとなる

#### 4.2.4 ブランドパワー分析カテゴリー（4モード）

**モード9: ブランドパワー分析①（セグメント×複数ブランド）**
- **目的**: 特定のセグメント内で、複数のブランドについて、ブランドパワー指標（BP1-BP4）を比較するモード
- **用途**: ブランドパワーの強み・弱みの可視化、競合他社とのパワーポジショニング比較
- **X軸**: ブランドパワー指標（固定4項目: BP1: 詳細認知, BP2: 知覚品質, BP3: ロイヤリティ, BP4: 話題性）
- **データ系列**: 選択した複数のブランド
- **フィルタ**: セグメント（単一選択）
- **データソース**: Excelシート（ブランドパワー関連列）
- **デフォルトチャート**: レーダーチャート（4指標の総合的なバランスを把握しやすい）

**モード10: ブランドパワー分析②（ブランド×複数セグメント）**
- **目的**: 単一ブランドにおける複数セグメント間のブランドパワー指標を比較
- **用途**: セグメント別のブランドパワー分析、属性別の強み・弱みの把握
- **X軸**: ブランドパワー指標（固定4項目: BP1-BP4）
- **データ系列**: 選択した複数のセグメント
- **フィルタ**: ブランド（単一選択）
- **デフォルトチャート**: レーダーチャート

**モード11: 将来性パワー分析①（セグメント×複数ブランド）**
- **目的**: 特定のセグメント内で、複数のブランドについて、将来性パワー指標（FP1-FP6）を比較
- **用途**: ブランドの将来的な成長ポテンシャルの評価、サステナビリティ・イノベーション力の可視化
- **X軸**: 将来性パワー指標（固定6項目: FP1-FP6）
- **データ系列**: 選択した複数のブランド
- **フィルタ**: セグメント（単一選択）
- **データソース**: Excelシート AJ～AO列
- **デフォルトチャート**: レーダーチャート

**モード12: 将来性パワー分析②（ブランド×複数セグメント）**
- **目的**: 単一ブランドにおける複数セグメント間の将来性パワー指標を比較
- **用途**: セグメント別の将来性評価、セグメント戦略の最適化
- **X軸**: 将来性パワー指標（固定6項目: FP1-FP6）
- **データ系列**: 選択した複数のセグメント
- **フィルタ**: ブランド（単一選択）
- **デフォルトチャート**: レーダーチャート

#### 4.2.5 アーキタイプ分析カテゴリー（2モード）

**モード13: アーキタイプ分析①（セグメント×複数ブランド）**
- **目的**: 特定のセグメント内で、複数のブランドについて、12タイプのアーキタイプ指標を比較
- **用途**: ブランドパーソナリティの評価、競合とのポジショニング比較、差別化戦略の立案
- **X軸**: アーキタイプ指標（固定12項目: 創造者、統治者、賢者、探検家、幼子、無法者、魔術師、英雄、恋人、道化師、仲間、援助者）
- **データ系列**: 選択した複数のブランド
- **フィルタ**: セグメント（単一選択）
- **データソース**: Excelシート AP～DE列（68項目から自動計算）
- **計算方法**: 各アーキタイプは対応するブランドイメージ項目の平均値として算出
- **デフォルトチャート**: レーダーチャート（12タイプの総合的なバランスを把握しやすい）

**モード14: アーキタイプ分析②（ブランド×複数セグメント）**
- **目的**: 単一ブランドにおける複数セグメント間の12タイプのアーキタイプ指標を比較
- **用途**: セグメント別のブランド認識評価、ターゲットセグメント選定、セグメント別コミュニケーション戦略の最適化
- **X軸**: アーキタイプ指標（固定12項目）
- **データ系列**: 選択した複数のセグメント
- **フィルタ**: ブランド（単一選択）
- **データソース**: Excelシート AP～DE列（68項目から自動計算）
- **計算方法**: 各アーキタイプは対応するブランドイメージ項目の平均値として算出
- **デフォルトチャート**: レーダーチャート

#### 4.2.3 過去比較モード（6種類・実装済み + 8種類・実装予定）

過去比較モードでは、最大3つのExcelファイルを読み込み、時系列でのデータ推移を分析する。

##### 実装済みモード（6種類）

**Mode 1: ファネル分析①（過去比較）**
- **ID**: `historical_funnel1_segment_brand`
- **目的**: 単一セグメント・単一ブランドのファネル①指標の時系列推移を分析
- **X軸**: ファネル①指標（FT, FW, FZ, GC, GJ, GL）
- **データ系列**: 各データソース（最大3つ）
- **セグメント**: 単一選択
- **ブランド**: 単一選択
- **デフォルトチャート**: 縦棒グラフ

**Mode 2: ファネル分析②（過去比較）**
- **ID**: `historical_funnel2_segment_brand`
- **目的**: 単一セグメント・単一ブランドのファネル②指標の時系列推移を分析
- **X軸**: ファネル②指標（T1, T2, T3, T4, T5）
- **データ系列**: 各データソース（最大3つ）
- **セグメント**: 単一選択
- **ブランド**: 単一選択
- **デフォルトチャート**: 縦棒グラフ

**Mode 3: ブランドイメージ分析（過去比較）**
- **ID**: `historical_brand_image_segment_brand`
- **目的**: 単一セグメント・単一ブランドのブランドイメージの時系列推移を分析
- **X軸**: ブランドイメージ項目TOP30（基準データソースで自動選定）
- **データ系列**: 各データソース（最大3つ）
- **セグメント**: 単一選択
- **ブランド**: 単一選択
- **デフォルトチャート**: 縦棒グラフ
- **特記事項**: 基準データソース（先頭）の数値降順でTOP30を自動選定

**Mode 4: ブランドパワー分析①（過去比較）**
- **ID**: `historical_brand_power_segment_brand`
- **目的**: 単一セグメント・単一ブランドの現在パワーの時系列推移を分析
- **X軸**: ブランドパワー指標（BP1, BP2, BP3, BP4）
- **データ系列**: 各データソース（最大3つ）
- **セグメント**: 単一選択
- **ブランド**: 単一選択
- **デフォルトチャート**: レーダーチャート

**Mode 5: ブランドパワー分析②（過去比較）**
- **ID**: `historical_future_power_segment_brand`
- **目的**: 単一セグメント・単一ブランドの将来性パワーの時系列推移を分析
- **X軸**: 将来性パワー指標（FP1, FP2, FP3, FP4, FP5, FP6）
- **データ系列**: 各データソース（最大3つ）
- **セグメント**: 単一選択
- **ブランド**: 単一選択
- **デフォルトチャート**: レーダーチャート

**Mode 6: アーキタイプ分析（過去比較）**
- **ID**: `historical_archetype_segment_brand`
- **目的**: 単一セグメント・単一ブランドのアーキタイプの時系列推移を分析
- **X軸**: アーキタイプ指標（12原型）
- **データ系列**: 各データソース（最大3つ）
- **セグメント**: 単一選択
- **ブランド**: 単一選択
- **デフォルトチャート**: レーダーチャート

##### 実装予定モード（8種類）

Mode 7-14は、詳細分析モードの残り8モードに対応する過去比較機能。設定駆動型アプローチにより、平均3-10分で実装可能。

##### データソース管理機能

**基本機能**
- **最大3ファイル**: 同時に読み込み可能
- **自動命名**: ファイル名から期間を自動抽出
  - 対応パターン: `YYYY_MM`, `YYYY-MM`, `YYYYMM`, `YYYYQ1-Q4`, `MonYYYY`等
  - 例: `BFD_202506.xlsx` → `2025年6月`
  - パターン不一致の場合: `データ1`, `データ2`, `データ3`
- **名前編集**: データソース名を手動変更可能（最大20文字、インライン編集）
- **ON/OFF切り替え**: データソースごとに表示/非表示を切り替え（最低1つは有効）
- **削除機能**: 個別削除 + 一括削除（リセットボタン）
- **リロード時クリア**: 画面リロード時に自動的にデータソースをクリア（メタデータのみLocalStorageから削除）

**UI/UX**
- ドラッグ&ドロップによるファイルアップロード
- データソース一覧表示（カード型UI）
- 基準データソース警告（Mode 3用）
- トグルスイッチによるON/OFF切り替え
- インライン編集による名前変更
- 確認ダイアログ付き削除ボタン

**隠しコマンド**
- **トリガー**: グローバルモードタブをCtrl+Shift+ダブルクリック
- **機能**: 3つのサンプルファイルを自動読み込み
  - `sample_202506.xlsx` → `2025年6月`
  - `sample_202406.xlsx` → `2024年6月`
  - `sample_202304.xlsx` → `2023年4月`
- **用途**: デモ環境のクイックセットアップ

### 4.3 データ可視化（グラフ描画）

#### 4.3.1 グラフタイプ
以下の4種類のグラフタイプを切り替え可能とする。
- **集合縦棒グラフ** (Bar Chart): 標準的な比較に適したグラフ
- **集合横棒グラフ** (Horizontal Bar Chart): データラベルの可読性が高いグラフ
- **折れ線グラフ** (Line Chart): トレンド分析に適したグラフ
- **レーダーチャート** (Radar Chart): 多項目の全体像把握に適したグラフ（モード9のデフォルト）

#### 4.3.2 表示指標
- **ファネル指標**: FT, FW, FZ, GC, GJ, GL（6項目）
- **時系列指標**: T1, T2, T3, T4, T5（5項目）

#### 4.3.3 インタラクション
- データラベルの表示/非表示切り替え
- ツールチップによる詳細数値の表示
- 凡例（Legend）の表示
- ホバー時のハイライト

### 4.4 UI/UX機能

#### 4.4.1 サイドバー操作
- 分析モードの切り替え（ドロップダウン）
- ファイルアップロード
- グラフ設定
- 対象データの選択
- モバイル対応（ハンバーガーメニューによる開閉）

#### 4.4.2 ドラッグ＆ドロップ並べ替え
- 比較対象のブランドやセグメントの表示順序をドラッグ＆ドロップで直感的に変更可能にする
- `@dnd-kit` ライブラリを使用した実装

#### 4.4.3 カラーテーマ
- プリセットされた10種類のカラーパレットから配色を選択可能にする
  - スタンダード (青・紫) - デフォルト
  - ビビッド (高コントラスト)
  - パステル (優しい色合い)
  - ビジネス (落ち着いた色)
  - モノクロ (グレースケール)
  - ネオン (蛍光色)
  - アース (ナチュラル)
  - オーシャン (ブルー系)
  - サンセット (オレンジ・ピンク系)
  - フォレスト (グリーン系)
- 選択されたブランド/セグメントに対して、一貫性のある色を自動割り当てする
- カラーインデックス管理により、選択を変更しても色が保持される

#### 4.4.4 タイプバッジ表示
各セクション（分析項目、セグメント、ブランド）に以下のバッジを表示：
- **【固定】**: 変更不可の固定値
- **【SA】**: Single Select（単一選択）
- **【MA】**: Multiple Select（複数選択）

### 4.5 エクスポート機能

#### 4.5.1 画像コピー
- 「グラフのみ」または「グラフ＋データテーブル」の画像をクリップボードにコピーする機能
- `html2canvas` ライブラリを使用した実装
- PNG形式、2倍スケール（高解像度）

#### 4.5.2 CSVエクスポート
- 現在表示されている分析データをCSV形式でダウンロードする機能
- BOM付与によりExcelでの文字化けを防止
- ファイル名にはタイムスタンプを付与（例: `funnel_analysis_20251126123456.csv`）
- 匿名化モードONの場合、CSV内のブランド名も匿名化される

## 5. 非機能要件

### 5.1 パフォーマンス

- 大量のデータを含むExcelファイルであっても、ブラウザ上で遅延なくパースおよび描画を行う
- Reactの再レンダリングを最適化し、UIのレスポンス性を維持する
- `useMemo` と `useCallback` を活用した不要な再計算の防止

### 5.2 セキュリティ

- **ローカル処理**: アップロードされたファイルはサーバーに送信せず、すべてブラウザ（クライアントサイド）上で処理する。外部へのデータ流出リスクを排除する
- **HTTPS推奨**: 本番環境ではHTTPS配信を推奨

### 5.3 ユーザビリティ

- レスポンシブデザインを採用し、PCおよびタブレット/スマートフォンでの閲覧に対応する
- エラーハンドリング：不正なフォーマットのファイルがアップロードされた場合、適切なエラーメッセージを表示する
- 直感的な操作性：ドラッグ&ドロップ、ワンクリック操作など

### 5.4 動作環境

- **対応ブラウザ**: モダンブラウザ (Chrome, Edge, Firefox, Safari) の最新版
- **必要環境**: JavaScript有効、LocalStorage利用可能

### 5.5 保守性・拡張性

- **設定駆動型アーキテクチャ**: 新しい分析モードを設定追加のみで実装可能
- **コンポーネント分割**: 責務が明確化され、保守が容易
- **TypeScript**: 型安全性により、リファクタリングが安全

## 6. 制約事項

### 6.1 データ形式の制約

- Excelファイルは特定のフォーマットに従う必要がある（詳細は `sample_excel_structure.md` を参照）
- 3行目がヘッダー、4行目以降がデータ、D列がブランド名という構造が必須

### 6.2 ブラウザの制約

- ブラウザのメモリ容量により、処理可能なデータサイズに上限がある
- 非常に大きなExcelファイル（数万行）の場合、パフォーマンスが低下する可能性がある

### 6.3 複数選択の上限

- ブランドおよびセグメントの複数選択は最大10個まで
- グラフの可読性を保つための制限

## 11. まとめ

BFD Analyticsは、Excelベースのブランドファネル調査データを直感的に可視化・分析するためのWebアプリケーションである。20の分析モード（詳細分析14種 + 過去比較6種実装済み + 8種予定）により多角的な分析が可能であり、設定駆動型アーキテクチャにより高い拡張性を持つ。

### 主要な要件のまとめ

- **20の分析モード**: 詳細分析14種 + 過去比較6種（実装済み）+ 8種（実装予定）
- **2つのグローバルモード**: 詳細分析 ⇔ 過去比較の切り替え
- **複数データソース管理**: 最大3ファイルの同時読み込み（過去比較モード）
- **171指標以上のサポート**: ファネル6 + 時系列5 + ブランドイメージ133 + ブランドパワー4 + 将来性パワー6 + アーキタイプ12 + その他
- **4種類のグラフタイプ**: 縦棒、横棒、折れ線、レーダー
- **クライアントサイド処理**: セキュアなデータ処理
- **匿名化機能**: デモ・プレゼンテーション対応
- **設定駆動型**: 容易な拡張性
- **自動計算機能**: アーキタイプ指標の自動算出
- **データソース管理**: ON/OFF切り替え、名前変更、一括削除

この要件定義に基づき、実装と検証が行われる。
